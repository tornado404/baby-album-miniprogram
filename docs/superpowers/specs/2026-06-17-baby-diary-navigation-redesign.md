## 0. 设计基础

### 0.1 Claymorphism 设计规范（必须遵循）

项目已有完整的 Claymorphism 设计系统，详见 [`miniprogram/styles/variables.wxss`](../../miniprogram/styles/variables.wxss)

**核心设计 Token：**

| 类别 | 变量名 | 十六进制值 | 用途 |
|------|--------|------------|------|
| 背景 | `--clay-bg` | `#fffbf8` | 页面主背景 |
| 主色 | `--clay-primary` | `#ffa87a` | 按钮、Tab 选中态 |
| 文字主色 | `--clay-text-primary` | `#4a4a4a` | 标题、正文 |
| 文字次色 | `--clay-text-secondary` | `#999999` | 副标题、描述 |
| 卡片白 | `--clay-card-white` | `#ffffff` | 卡片背景 |
| 阴影 | `--clay-shadow-card` | `0px 6px 16px 0px rgba(230, 198, 179, 0.35)` | 通用卡片阴影 |
| 圆角 | `--clay-radius-md` ~ `--clay-radius-xxl` | `16rpx` ~ `24rpx` | 圆角范围 |
| 字体 | `--clay-font-*` | `20rpx` ~ `40rpx` | 字号层级 |

**设计稿要求：**
- 设计稿宽度：375px（对应小程序 750rpx）
- 圆角统一使用：16rpx / 18rpx / 20rpx / 24rpx
- 阴影统一使用上述变量值
- 颜色严格使用十六进制值（不要用变量引用，便于开发直接复制）

## 5. 交互状态规范

| 场景 | 状态 |
|------|------|
| 按钮点击 | 50ms 内反馈，轻微缩放或变色 |
| 卡片点击 | 背景色变深 5%，50ms 过渡 |
| 选中态 | 背景高亮，边框/阴影强化 |
| 加载态 | skeleton 骨架屏或 `--t-loading` 组件 |
| 错误态 | 红色提示 toast |
| 空状态 | 插画 + 引导文案 + 行动按钮 |

---

## 6. 权限规则

| 规则 | 说明 |
|------|------|
| 操作按钮可见性 | 仅 `record.uploader_id === current_user.id` 时显示 ⋮ |
| 家人共享 | 管理员可操作全部，普通成员仅自己的记录 |
| 精选/隐藏 | 状态字段 `is_featured` / `is_hidden` |
