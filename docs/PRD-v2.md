## 1. 概述

### 1.1 本版本目标

v1.x 完成了基础 MVP：图片上传、时间轴浏览、宝宝档案管理。v2.0 在此基础上增强**数据丰富度、交互流畅性、内容管理能力**。

### 1.2 核心改进点

| 改进 | 说明 | 优先级 |
|------|------|--------|
| **上传增强** | 上传时自动获取 GPS 位置、添加标签、时刻描述、里程碑标记 | P0 |
| **相册管理** | 批量选择、归档/取消归档、标签筛选、位置筛选 | P0 |
| **底部 Tab 重构** | 拆分「首页」「相册」为「成长历程」「素材库」，各司其职 | P0 |
| **媒体查看器** | 点击卡片进入全屏查看器（暗色背景、滑动切换、信息叠加） | P0 |
| **GPS 位置服务** | 从 EXIF 读取 GPS + 手机 GPS 获取 + 地图展示 | P1 |

### 1.3 用户旅程变化

```
v1.x:                          v2.0:
上传 → 返回列表 → 查看卡片      上传 → 添加位置/标签/时刻 → 待归档
                                 ↓
首页/相册 → 同一页面            素材库（草稿）→ 归档 → 成长历程（对外）
                                 ↓
点击卡片 → 无响应               点击卡片 → 全屏查看器（暗色背景+信息+滑动）
```

## 2. 底部 Tab 重构

### 2.1 当前问题

当前底部导航 4 个 Tab：
```
🏠 首页 → album_home（时间轴+媒体卡片）
📖 相册 → album_home（同一页面，不同参数）
➕ 上传 → upload
👤 我的 → settings
```

`首页`和`相册`实际上是同一个页面，用户无法区分「对外展示的成长记录」和「未处理的原始素材」。

### 2.2 新 Tab 结构

```
👶 成长历程 → journey（对外可见的已归档媒体，精美时间轴）
📦 素材库  → gallery（未归档媒体，列表/网格模式，批量管理工具）
➕ 上传    → upload（不变）
👤 我的    → settings（不变）
```

### 2.3 Tab 对比

| Tab | 路由 | 可见性 | 内容 | 交互 |
|-----|------|--------|------|------|
| 👶 成长历程 | `/pages/journey/journey` | **对外可见**（分享给家人） | 已归档的媒体，带位置/时刻/里程碑 | 精美时间轴滚动、卡片点击查看详情 |
| 📦 素材库 | `/pages/gallery/gallery` | **仅自己可见** | 所有未归档媒体 | 网格/列表切换、批量选择、归档/标签编辑 |
| ➕ 上传 | `/pages/upload/upload` | — | 上传流程（增强） | 上传后弹出编辑浮层（位置/标签/时刻） |
| 👤 我的 | `/pages/settings/settings` | — | 设置 | 不变 |

### 3.2 GPS 位置获取策略

| 来源 | 优先级 | 说明 |
|------|--------|------|
| EXIF GPS | 最高 | 相机/手机拍摄时嵌入，`wx.getImageInfo` 可读取 |
| 手机 GPS | 次高 | 选择照片后调用 `wx.getLocation` 获取当前位置 |
| 手动输入 | 兜底 | 用户可手动搜索/选择位置 |

```typescript
async function getPhotoLocation(filePath: string): Promise<Location | null> {
  // 1. 尝试从 EXIF 读取 GPS
  try {
    const info = await wx.getImageInfo({ src: filePath });
    if (info.gpsLongitude && info.gpsLatitude) {
      return { lat: info.gpsLatitude, lng: info.gpsLongitude, source: 'exif' };
    }
  } catch (e) {}

  // 2. 尝试手机 GPS
  try {
    const loc = await wx.getLocation({ type: 'wgs84' });
    return { lat: loc.latitude, lng: loc.longitude, source: 'gps' };
  } catch (e) {}

  return null;
}
```

## 7. 数据模型扩展

### 7.1 Media 模型扩展

```typescript
// miniprogram/typings/models/media.ts - 扩展

interface Media {
  id: string;
  babyId: string;
  type: 'image' | 'video' | 'threedmodel';
  title: string;

  // v2.0 新增字段
  /** GPS 位置 */
  location?: {
    latitude: number;
    longitude: number;
    name: string;        // 中文地名（如"北京市朝阳区"）
    source: 'exif' | 'gps' | 'manual';
  };

  /** 标签列表 */
  tags?: string[];

  /** 美好时刻文本 */
  moment?: string;

  /** 宝宝飞跃阶段 */
  milestone?: MilestoneStage;

  /** 归档状态 */
  isArchived: boolean;   // true=已归档（对外可见）/ false=未归档

  /** 归档时间 */
  archivedAt?: string;

  // 已有字段...
  url: string;
  thumbnailUrl: string;
  captureDate: string;
  createdAt: string;
}

type MilestoneStage =
  | 'newborn'     // 🍼 新生儿
  | 'head_up'    // 👶 抬头期
  | 'explore'    // 👐 探索期
  | 'teething'   // 🦷 出牙期
  | 'toddler'    // 🚶 学步期
  | 'walking'    // 🏃 行走期
  | 'language'   // 🗣️ 语言期
  | 'creative'   // 🎨 创造期
  ;
```
