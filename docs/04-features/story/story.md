# 宝宝成长相册 - 产品故事文档

## 1. 产品愿景与使命

### 1.1 产品简介

**宝宝成长相册**是一款微信小程序，致力于为新手父母提供一个温馨、便捷的方式来记录和见证宝宝的成长历程。我们相信，每一个宝宝的成长瞬间都值得被珍藏，而技术的力量可以让这些珍贵的记忆以更加生动和立体的方式被保存和回顾。

### 1.2 核心价值主张

- **记录真实**：捕捉宝宝成长中的每一个重要时刻，从第一次微笑到蹒跚学步
- **时光胶囊**：将记忆以时间轴的形式串联，让父母能够重温宝宝成长的每一个阶段
- **三维重现**：通过Web3D技术，将宝宝的可爱模样以立体的形式永久保存
- **简单分享**：一键上传，轻松整理，让记录变得毫无负担

### 1.3 产品使命

让每一位父母都能轻松、专业地记录宝宝的成长历程，将那些转瞬即逝的美好时刻转化为永恒的记忆资产。

### 1.4 设计理念

- **温暖而专业**：视觉风格温馨友好，同时体现专业品质
- **以家庭为中心**：所有功能设计都围绕家庭场景展开
- **隐私优先**：宝宝的影像资料属于家庭，保护用户隐私是产品的基石
- **情感连接**：通过时间轴和回忆展示，增强家庭成员之间的情感纽带

## 2. 目标用户

### 2.1 主要用户群体

#### 2.1.1 新手父母（Primary User）

- **年龄**：25-35岁
- **特征**：
  - 首次为人父母，对宝宝成长记录有强烈需求
  - 具备一定的智能手机使用经验
  - 活跃于社交媒体，愿意分享宝宝成长点滴
  - 对新兴技术（如3D模型）有一定接受度
- **核心诉求**：
  - 便捷地记录宝宝成长瞬间
  - 整理和归纳大量照片和视频
  - 以时间轴形式回顾成长历程
  - 与家人分享宝宝成长动态

#### 2.1.2 祖父母辈（Secondary User）

- **年龄**：50-65岁
- **特征**：
  - 希望了解宝宝的成长动态
  - 传统相册的使用习惯
  - 可能对复杂操作不太熟悉
- **核心诉求**：
  - 简单直观地查看宝宝照片
  - 接收子女分享的宝宝成长更新
  - 轻松保存和收藏宝宝照片

### 2.2 用户场景

#### 场景一：日常记录
> 宝宝今天学会翻身了，妈妈用手机拍下这个珍贵瞬间，一键上传到成长相册中，自动按时间归档。

#### 场景二：成长回顾
> 周末家庭聚会，爷爷想看看宝宝这三个月的变化，打开相册，时间轴清晰展示每次里程碑记录。

#### 场景三：3D留念
> 宝宝满月了，专业摄影师拍摄了多角度照片，通过小程序生成3D模型，永久保存宝宝满月时的模样。

#### 场景四：家人共享
> 远在海外的舅舅想看小侄女的近照，妈妈将最近的成长记录分享给舅舅，舅舅可以随时查看。

## 3. 核心功能

### 3.1 媒体类型支持

#### 3.1.1 图片上传与管理

- **支持格式**：JPEG、PNG、WebP
- **功能特性**：
  - 单张或批量上传
  - 自动压缩优化，节省存储空间
  - EXIF信息提取（拍摄时间、地理位置等）
  - 图片预览和缩放
  - 图片删除和编辑时间戳

#### 3.1.2 视频记录

- **支持格式**：MP4（微信小程序限制）
- **功能特性**：
  - 视频时长限制：建议不超过3分钟
  - 视频压缩和转码
  - 封面自动提取或手动选择
  - 视频播放和全屏查看
  - 支持静音播放（默认）

#### 3.1.3 Web3D模型展示

- **技术实现**：基于Three.js的WebGL渲染
- **模型格式**：GLTF/GLB（ glTF 2.0标准）
- **功能特性**：
  - 360度旋转查看
  - 缩放操作
  - 模型材质展示
  - 灯光效果调整
  - 截图保存功能
- **应用场景**：
  - 满月照、百日照等重要节点的3D纪念
  - 专业摄影师提供的3D扫描数据展示
  - 宝宝可爱表情的3D还原

### 3.2 时间轴展示系统

#### 3.2.1 时间维度定义

- **按月龄展示**：
  - 以宝宝的月龄为主要时间单位
  - 显示格式：「X个月Y天」
  - 自动计算宝宝当前月龄
- **里程碑标记**：
  - 翻身、爬行、走路、说话等关键节点
  - 自定义里程碑添加
  - 里程碑日期自动标注

#### 3.2.2 时间轴交互

- **滑动浏览**：
  - 垂直滚动浏览时间轴
  - 滑动流畅，无卡顿
  - 惯性滚动效果
- **快速定位**：
  - 月龄选择器跳转
  - 点击顶部月龄快速回到顶部
  - 滑动速度检测，智能切换浏览模式

### 3.3 排序与展示逻辑

#### 3.3.1 倒序排列（默认）

- 最新上传的内容显示在最上方
- 用户打开应用首先看到最新的成长记录
- 符合移动端用户"看最新内容"的使用习惯

#### 3.3.2 默认定位

- 应用启动时自动定位到最新内容位置
- 无需用户手动操作即可看到宝宝最近动态
- 顶部固定显示当前宝宝月龄信息

### 3.4 瀑布流布局

#### 3.4.1 瀑布流设计

- **自适应列数**：
  - 根据屏幕宽度自动调整列数
  - 竖屏设备显示2-3列
  - 横屏设备自动切换为3-4列
- **高度计算**：
  - 根据媒体内容原始比例计算显示高度
  - 图片和视频统一瀑布流展示
  - 3D模型以固定比例卡片形式展示

#### 3.4.2 滚动性能优化

- **懒加载机制**：
  - 仅加载可视区域内的内容
  - 滚动时动态加载上下区域内容
  - 减少初始加载数据量
- **图片优化**：
  - 缩略图预览
  - 渐进式加载
  - 缓存机制

#### 3.4.3 交互反馈

- **下拉刷新**：获取最新内容
- **上拉加载更多**：历史内容自动加载
- **加载状态指示**：骨架屏或loading动画

## 4. 用户体验目标

### 4.1 易用性（Ease of Use）

#### 4.1.1 学习成本

- 新用户首次使用即可快速上手
- 核心操作路径不超过3步
- 操作引导清晰，新手教程友好

#### 4.1.2 操作效率

- 从打开应用到看到内容不超过3秒
- 单张照片上传到显示不超过2秒
- 批量上传支持后台处理，不阻塞操作

#### 4.1.3 容错性

- 误操作可撤销
- 网络异常自动重试
- 意外退出数据自动保存

### 4.2 情感化设计（Emotional Engagement）

#### 4.2.1 温暖视觉

- 柔和的配色方案（以暖色调为主）
- 圆润的UI元素设计
- 可爱的吉祥物或插画元素
- 留白充足，不造成视觉疲劳

#### 4.2.2 情感共鸣

- 里程碑节点的特别提示
- 成长日记的回忆回顾功能
- 生日、节日等特殊时点的氛围包装

#### 4.2.3 仪式感

- 重要节点的专属纪念卡片
- 成长对比功能（before/after）
- 成就徽章系统（记录数量里程碑）

### 4.3 视觉吸引力（Visual Appeal）

#### 4.3.1 界面美学

- 现代简洁的设计风格
- 精致的图标和插画
- 优雅的动画过渡
- 和谐的色彩搭配

#### 4.3.2 媒体展示

- 图片展示清晰、色彩还原准确
- 视频播放流畅
- 3D模型渲染精美
- 全屏沉浸式浏览体验

### 4.4 性能体验（Performance）

- 首屏加载时间 < 3秒
- 页面切换流畅，无卡顿
- 内存占用合理，不影响手机正常使用
- 耗电量低

## 6. 第一阶段（Phase 1）产品范围

### 6.1 阶段目标

第一阶段聚焦于核心功能的实现和用户体验的验证，建立稳定的产品基础，为后续迭代提供数据和反馈支撑。

### 6.2 功能范围

#### 6.2.1 宝宝信息管理

- **宝宝档案创建**
  - 宝宝姓名（昵称）
  - 宝宝性别
  - 宝宝生日
  - 宝宝头像设置
  - 预产期/实际出生日期选择

- **多宝宝支持**
  - 支持添加多个宝宝档案
  - 宝宝快速切换
  - 每个宝宝独立的成长相册

#### 6.2.2 媒体上传

- **图片上传**
  - 单张图片上传
  - 批量图片上传（最多9张）
  - 从相册选择
  - 拍照上传
  - 上传进度显示
  - 失败重试机制

- **视频上传**
  - 从相册选择视频
  - 视频时长限制提示
  - 封面选择
  - 上传进度显示

- **3D模型上传**
  - 支持GLTF/GLB格式
  - 模型文件大小限制（建议<10MB）
  - 上传进度和状态提示

#### 6.2.3 时间轴展示

- **核心时间轴**
  - 按月龄展示内容
  - 倒序排列（最新在上）
  - 月龄标题固定吸顶
  - 平滑滚动体验

- **里程碑系统**
  - 自动识别常见里程碑（翻身、爬行等）
  - 自定义里程碑添加
  - 里程碑日期标注

#### 6.2.4 瀑布流浏览

- **瀑布流布局**
  - 2列瀑布流展示
  - 自适应内容高度
  - 图片/视频/3D模型统一展示

- **性能优化**
  - 图片懒加载
  - 缩略图预览
  - 分页加载（每页20条）

#### 6.2.5 基础用户功能

- **微信授权登录**
  - 微信一键登录
  - 获取用户基本信息

- **本地存储**
  - 使用微信云存储
  - 媒体文件云端同步

### 6.3 界面范围

| 页面 | 说明 | 优先级 |
|------|------|--------|
| 首页/时间轴 | 宝宝成长时间轴展示 | P0 |
| 上传页 | 媒体上传入口 | P0 |
| 宝宝档案页 | 宝宝信息管理 | P0 |
| 内容详情页 | 单条内容查看 | P1 |
| 3D模型查看页 | 3D模型展示 | P1 |
| 我的/设置页 | 用户设置 | P2 |

### 6.4 技术范围

- **前端框架**
  - 微信小程序Skyline渲染引擎
  - glass-easel组件框架
  - TypeScript开发
  - Vant Weapp组件库

- **数据存储**
  - 微信云开发
  - 云存储（媒体文件）
  - 云数据库（用户和内容元数据）

- **第三方集成**
  - 3D模型渲染：Three.js
  - 图片处理：WeChat Mini Program Canvas API

### 6.5 非功能范围（Phase 1不考虑）

- 社交分享功能
- 家人邀请/协作功能
- 高级数据分析
- 打印/实物相册制作
- AI智能识别（宝宝表情、动作）
- 小程序广告变现
