## 三、风险评估

### 3.1 技术风险

| 风险项 | 风险等级 | 描述 | 缓解策略 |
|--------|----------|------|----------|
| Skyline/glass-easel 兼容性 | **高** | 渲染引擎与 Vant Weapp 组件的兼容性问题 | 1. 优先验证 Vant 组件在 Skyline 下的表现<br>2. 准备 fallback 方案（原生组件） |
| Vant Weapp 适配 | **中** | Vant 组件在 Skyline 下的样式和交互差异 | 1. 参考 Vant 官方兼容性文档<br>2. 组件添加 `styleIsolation: apply-shared` |
| 瀑布流布局算法 | **中** | 懒加载时高度计算不准导致布局跳动 | 1. 使用固定占位高度<br>2. 图片预加载获取真实尺寸 |
| 图片压缩兼容性 | **低** | `wx.compressImage` API 存在平台差异 | 使用 Vant uploader 的压缩能力作为替代 |
| Web3D 模型支持 | **低** | Phase 1 不包含，但文档提及 | 预留接口，Phase 2 再集成 Three.js |

### 3.2 依赖风险

| 风险项 | 风险等级 | 描述 | 缓解策略 |
|--------|----------|------|----------|
| T-03 依赖瓶颈 | **高** | T-03 依赖 T-05 和 T-08，两者完成才可开始 | v0.1 优先完成 T-05 和 T-08，T-03 尽早介入 |
| 任务边界模糊 | **中** | T-03 包含多个子功能，可能超时 | 拆分为子任务，设置明确的范围控制 |
| 测试阻塞 | **中** | T-09 依赖所有任务，任何延迟都影响测试 | 每日集成，持续验证 |

### 3.3 复杂度风险

| 风险项 | 风险等级 | 描述 | 缓解策略 |
|--------|----------|------|----------|
| 瀑布流布局性能 | **高** | 双列布局 + 懒加载 + 分页，复杂度较高 | T-05 独立开发测试，尽早验证核心算法 |
| 多组件协同 | **中** | 首页依赖多个组件，协调成本高 | 定义清晰的组件接口协议 |
| 月龄计算逻辑 | **低** | 涉及闰年、日期边界等 | 使用成熟的 date-fns 或 moment.js |

## 四、建议

### 4.1 最佳迭代顺序

```
第1步: T-01 → T-02 (串行，基础中的基础)
            ↓
第2步: 并行执行
       ├─ T-05 (瀑布流组件，独立可测)
       └─ T-08 (存储服务，独立可测)
            ↓
第3步: T-03 (首页，需 T-05 + T-08 完成)
            ↓
第4步: 并行执行
       ├─ T-04 (上传组件)
       ├─ T-06 (详情页)
       └─ T-07 (筛选功能)
            ↓
第5步: T-09 (集成测试)
```

### 4.2 关键检查点

| 里程碑 | 完成条件 | 验证方法 |
|--------|----------|----------|
| **CP1: 架构完成** | T-01, T-02, T-05, T-08 | 编译通过，单元测试通过 |
| **CP2: 首页可用** | T-03 完成，瀑布流+存储对接完成 | 手动功能测试 |
| **CP3: 功能完备** | T-04, T-06, T-07 完成 | 功能测试用例执行 |
| **CP4: 版本发布** | T-09 完成，所有已知 Bug 修复 | 回归测试通过 |

### 4.3 任务拆分调整建议

#### 建议一：拆分 T-03（相册首页开发）

当前 T-03 工时 8h，依赖最多，建议拆分为：

| 子任务 | 描述 | 依赖 |
|--------|------|------|
| T-03a | 首页框架 + 宝宝选择器 | T-01, T-02 |
| T-03b | 瀑布流集成 | T-03a, T-05 |
| T-03c | 上传入口 + 筛选入口 | T-03a, T-08 |
| T-03d | 下拉刷新 + 上拉加载 | T-03b |

#### 建议二：提前验证技术风险

在正式开发前，增加 **技术验证任务** (建议 2h):

| 验证项 | 验证方法 | 风险等级 |
|--------|----------|----------|
| Skyline + Vant NavBar | 原型测试导航栏 | 高 |
| Skyline + Vant Tabs | 原型测试标签切换 | 中 |
| 瀑布流布局算法 | 原型测试不同图片比例 | 中 |
| 图片压缩 API | 原型测试压缩效果 | 低 |

#### 建议三：T-09 改为持续集成

不建议将 T-09 作为一个独立阶段，建议改为：

1. 每日构建，每次提交触发编译检查
2. 每个任务完成后执行该任务的单元测试
3. v0.3 阶段进行完整的集成测试和性能测试

## 五、总结

### 5.1 可行性结论

**渐进迭代可行性**: **可行**，需注意以下要点：

1. **关键路径明确**: T-01 → T-02 → T-08 → T-09 是核心链路，必须优先完成
2. **并行空间充足**: T-05 可与其他任务完全并行，节省 5h 工时
3. **风险可控**: 主要技术风险（Skyline/Vant 兼容性）可通过提前验证降低
4. **迭代周期合理**: 三个版本可在 9-12 天内完成

### 5.2 关键成功因素

1. **尽早验证 Skyline + Vant 兼容性**（影响瀑布流和首页开发）
2. **瀑布流组件独立可测**（T-05 是关键里程碑）
3. **存储服务接口稳定**（T-08 是多个任务的前置依赖）
4. **持续集成，不把问题留到最后**

### 5.3 调整优先级建议

| 任务 | 原优先级 | 建议优先级 | 理由 |
|------|----------|-----------|------|
| T-05 瀑布流组件 | P1 | **P0** | 首页依赖，是关键路径 |
| T-08 本地存储服务 | P0 | **维持 P0** | 多个任务依赖 |
| T-07 月龄筛选功能 | P2 | **P1** | 核心用户体验功能 |
